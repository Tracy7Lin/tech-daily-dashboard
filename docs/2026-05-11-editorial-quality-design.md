# 科技日报编辑质量增强设计

## 目标

把当前日报从“规则化聚合器”进一步提升到：

- `首页总览 + 主题对比` 更像轻量行业分析博客
- `公司动态 + 单条新闻` 保持高质量新闻简报写法

本轮不追求生成长篇评论，而是补一个独立的编辑层，让系统能对已有结构化内容做更强的组织、比较与归纳。

## 现状问题

当前展示层信息量已经提升，但内容理解仍偏机械：

- 首页 headline 更像统计说明，不像编辑判断
- 主题卡中的 `summary / comparison / trend` 仍然过于模板化
- `其他重要动态` 缺乏解释，难以看出为什么被放在一起
- 单条摘要已经可读，但跨公司意义、节奏变化、主次关系表达不够

## 设计原则

- 新增 `editorial` 层，不把表达逻辑塞回 `topics.py` 或 `pipeline.py`
- 继续保持 `summarizer` 只处理单条新闻
- 主题聚类逻辑尽量不动，本轮重点是“写法”和“解释力”
- 输出必须稳定、可测试，不能依赖随意发挥

## 架构调整

新增模块：

- `src/tech_daily/editorial.py`

职责：

- 生成分析型日报 headline
- 为主题簇生成更像编辑写法的 `summary / comparison / trend`
- 统一管理“公司切入点”的判定词表与表达逻辑

原有模块边界：

- `classifier.py`：标签与重要度
- `summarizer.py`：单条简报摘要
- `topics.py`：聚类与排序
- `editorial.py`：分析表达
- `pipeline.py`：编排调用

## 方案

### 1. 首页总览改成分析型 headline

从原来的：

- 今天共筛出多少条
- 热门主题是什么

升级为：

- 今天真正的核心信号是什么
- 为什么这两个主题值得关注
- 活跃公司在各自推进什么方向

目标效果：

- 不是纯统计说明
- 读者看完 headline 就知道“今天的产业节奏”

### 2. 主题对比改成编辑表达

每个主题簇输出三层信息：

- `summary`：这一主题今天到底发生了什么
- `comparison`：各公司切入点差异
- `trend`：这说明行业正在往哪里演进

表达上避免：

- “相关公司包括 X，显示该方向正在继续演进”
- “A 偏 product；B 偏 technology” 这类原始标签句

改为更接近：

- OpenAI 把重点放在模型与平台能力同步外放，Google 更偏具体产品与分发场景落地。
- 这说明竞争正在从单点能力比拼转向“能力 + 入口 + 落地”的组合战。

### 3. 统一公司切入点词汇

引入独立的切入点映射，不再直接泄露内部 category/tag 名称。

例如：

- `model + developer` -> `模型与平台能力外放`
- `customer + enterprise` -> `客户落地与业务验证`
- `safety + consumer` -> `面向用户触点的安全控制`
- `product + consumer` -> `具体产品与使用场景落地`
- `infrastructure` -> `底层基础设施扩张`

### 4. 单条简报维持简报写法

本轮不重写整套 `summarizer`，只保留它继续服务公司卡与重点观察。

如后续继续优化，可以再独立推进：

- 标题驱动的动作识别
- 更自然的中文句式
- 更稳的“为什么重要”归纳

## 验收标准

- 首页 headline 不再以“今天共筛出 X 条”作为主要表达
- 主题卡的 `summary / comparison / trend` 读起来更像编辑判断
- `comparison` 中不再裸露 `product / technology / general`
- 单元测试能覆盖新的 editorial 输出

## 非目标

- 本轮不做 LLM 生成
- 本轮不重做 topic 聚类算法
- 本轮不引入新的数据存储结构
