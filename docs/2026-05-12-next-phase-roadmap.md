# 下一阶段路线图

## 目标

当前项目已经具备：

- 可生成的日报主链路
- LLM 混合表达层
- 本地静态站点输出
- Windows 本地自动化入口

下一阶段不再追求“大而全”，而是按批次推进，优先提高系统的长期可运行性与可维护性。

## 优先级顺序

### 第一层：自动化与可观测性

优先原因：

- 这是最接近日常使用体验的能力
- 可以在不改抓取和内容逻辑的前提下，提升稳定性

本批次包含：

- `generate-today` CLI 入口
- `health-check` 环境健康检查
- PowerShell 本地自动化脚本
- 日志落盘与运行摘要输出

当前状态：

- 已完成

### 第二层：内容源可靠性

目标：

- 降低“当日内容过空”概率
- 提高 HTML 官方源的日期与正文提取稳定性

建议任务：

- 精修重点 HTML 源的发布时间抽取
- 补充低产出公司的官方源入口
- 优化源状态说明文案，让失败原因更可读

### 第三层：工程化与运维

目标：

- 让仓库更容易长期维护

建议任务：

- 增加 `dry-run` 模式
- 增加更细的配置校验
- 增加更稳定的集成或快照级验证
- 补充运维与恢复说明

### 第四层：前端体验收尾

目标：

- 提升长期阅读体验

建议任务：

- modal 焦点管理与可访问性
- 结论区 / 证据区分层
- 移动端密度与留白微调

### 第五层：Agent 预备层

目标：

- 为未来科技情报分析 agent 预留可复用边界

建议任务：

- 固化 `brief_generation`
- 固化 `topic_comparison`
- 固化 `daily_editorial`
- 为跨日追踪增加更稳定的数据接口

## 当前推荐下一批

如果继续按务实顺序推进，推荐下一批做：

1. `source reliability`
2. `dry-run + config validation`
3. `health-check` 继续增强为更细粒度诊断
