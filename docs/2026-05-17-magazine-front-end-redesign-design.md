# 科技情报杂志式前端改造设计

日期：2026-05-17  
范围：`tech-daily-dashboard` 前端信息架构与视觉语言改造  
状态：设计已确认，待进入实现计划

## 1. 背景

当前项目已经具备较完整的内容与 agent 能力链：

- 主日报生成
- 当日情报判断
- 跨日观察
- 专题跟踪
- 主题档案
- 前端对话 agent

但前端仍然存在两个结构性问题：

1. 页面承担了过多层级的阅读任务。  
   首页、日报页、深度分析区块、聊天入口大量堆叠在单页结构里，虽然信息完整，但缺少明显的阅读分层。

2. 视觉语言偏“成熟产品面板”，缺少“刊物封面”和“专题栏目”的表现感。  
   当前界面清爽、稳定、可用，但情绪张力、记忆点和编辑感仍然不足，难以形成独特的科技情报产品气质。

本次改造不以增加新 agent 层为目标，而以**重做前端信息架构与视觉表达**为目标。

## 2. 目标

本次前端改造目标如下：

1. 把当前站点从“单页信息面板”升级为“科技情报杂志”。
2. 建立清晰的页面分工，不再把所有内容塞在一个页面里。
3. 让首页具备更强的封面感、表现力和品牌气质。
4. 保持科技感、简约感与快速提取信息的能力，不为了艺术感牺牲可读性。
5. 让现有的 `theme_tracking`、`theme_dossier`、`chat agent` 等能力，在前端上拥有更合理的位置和更高的被使用价值。

## 3. 非目标

本次不做以下事情：

- 不重写抓取器、分类器或 LLM 内容生成链路
- 不新增新的 agent 产品层
- 不引入复杂的前端框架或单页应用架构
- 不做赛博朋克化的高噪音视觉特效
- 不做跨页面持久化用户状态系统

## 4. 核心方向

本次改造采用：

**持续更新的主刊首页 + 每日一期日报 + 深度专题页 + 主题档案页**

并将整体视觉风格收束为：

**封面式科技情报杂志**

这意味着：

- 首页不是普通导航页，而是主刊封面
- 日报页不是全站总页，而是“当天正文”
- 深度分析内容不再堆在日报页内，而是拆出去形成栏目页
- 聊天抽屉不再只是工具入口，而要更像研究助理边栏

## 5. 信息架构

### 5.1 主刊首页

主刊首页是长期稳定入口，不依赖某一天是否刚好有很多日报内容。

首页职责：

- 作为站点封面
- 以“当前主专题”为核心主视觉
- 承接主线判断、主专题、专题入口与最近几期导航
- 建立刊物感和品牌感

首页应包含：

- 站点刊名与简短说明
- 当前主专题封面区
- 一句编辑判断
- 主专题摘要 / 为什么值得看
- 进入专题页
- 查看最新日报
- 查看主题档案
- 最近几期日报入口

首页不应承载：

- 完整公司动态列表
- 全量跨日观察正文
- 全量 dossier 正文
- 过多下钻内容

### 5.2 每日一期日报页

日报页回归“当期正文”的角色。

日报页职责：

- 回答“今天发生了什么”
- 提供快速扫读
- 展示当天重点、公司动态和主题概览

日报页保留：

- headline
- 重点观察
- 公司动态
- 快速主题概览
- chat 入口

日报页弱化或移出：

- 跨日观察全文
- 专题跟踪全文
- 主题档案全文

这些深度内容保留简短入口或摘要，而不继续堆在主正文中。

### 5.3 专题页

专题页用于承载“最近几天正在发生什么”。

专题页职责：

- 汇总跨日观察
- 汇总专题跟踪
- 展示主题档案摘要
- 形成研究栏目页

专题页回答：

- 这个主题最近几天怎么演化
- 哪些公司持续参与
- 各自切入点有什么差异
- 明天是否还值得继续跟踪

### 5.4 主题档案页

主题档案页用于承载主题 dossier 的沉淀内容。

档案页职责：

- 主题定义
- 当前状态
- 公司位置
- 关键时间线
- 跟踪决策
- 下一步关注

档案页更接近：

- 专题档案页
- 小型研究卡
- 未来长期情报记忆入口

## 6. 页面关系

推荐的阅读路径：

1. 用户进入主刊首页  
2. 从主专题进入专题页  
3. 需要当日细节时进入日报页  
4. 需要更沉淀的主题信息时进入主题档案页  
5. 在任一页面通过 chat 进行研究式追问

这形成如下结构：

- 首页：主刊封面
- 日报页：当期正文
- 专题页：深度栏目
- 档案页：沉淀档案
- Chat：全局研究助理

## 7. 首页封面设计

首页采用三段式封面结构。

### 7.1 封面主视觉

第一屏服务于一个核心对象：`当前主专题`

内容包括：

- 刊名
- 主专题标题
- 一句编辑式主题判断
- 一段简短主题定义
- 两个主按钮：
  - `进入专题页`
  - `查看最新日报`

要求：

- 强标题
- 大留白
- 更少碎信息
- 明显主视觉焦点

### 7.2 封面副栏

封面旁边提供一条窄而精的编辑导读栏。

建议信息：

- 今日一句判断
- 当前阶段
- 参与公司
- 下一步关注

其角色不是 KPI 面板，而是封面导读。

### 7.3 栏目入口区

封面下方进入刊物栏目区，而不是正文堆叠区。

建议栏目入口：

- 最新日报
- 专题页
- 主题档案
- 最近几期日报

要求：

- 栏目化
- 可点击进入
- 以摘要和视觉主次为主，不展开正文

## 8. 视觉语言

### 8.1 总体气质

整体视觉定性为：

**科技情报杂志**

它需要兼顾：

- 科技感
- 编辑感
- 信息秩序
- 品牌记忆点

避免：

- 过度 SaaS 化
- 过度后台化
- 过度赛博朋克化
- 过度复古杂志化

### 8.2 排版

排版策略：

- 首页标题更像封面标题
- 栏目标题层级更强
- 正文维持克制
- 提高标题与导语的节奏差异

预期效果：

- 更强的“编辑感”
- 更清晰的主次关系
- 更少“一切都像同一种 section”的问题

### 8.3 构图与留白

构图策略：

- 首页第一屏更大留白
- 主视觉区与栏目入口区明显分层
- 重点卡允许更强的非对称构图
- 次级区块避免平均分布导致的“面板感”

### 8.4 背景与材质

保留当前的轻科技底色，但增加更明显的层次：

- 更具空间感的背景场
- 局部高光
- 局部描边
- 主卡、栏目卡、入口卡之间的材质差异

要求是：

- 精致
- 克制
- 有层次
- 不干扰阅读

## 9. 导航与动效

### 9.1 导航结构

需要新增更明确的站内导航，而不是只靠单页滚动或少量返回链接。

建议导航层级：

- 首页
- 最新日报
- 专题
- 档案

导航形态可以采用：

- 顶部轻导航
- 栏目入口卡
- 页内辅助 breadcrumb

### 9.2 动效原则

动效目标是提高表现力和栏目感，而不是增加热闹感。

允许动效：

- 页面 reveal
- 卡片 hover 提示
- 导航入口的方向性位移
- 轻量滚动节奏
- 聊天抽屉开合强化“研究助理”身份

避免动效：

- 大面积持续运动
- 粒子类噪音特效
- 明显拖慢阅读的动画
- 过度弹跳、夸张 easing

### 9.3 动效作用

动效只承担两件事：

1. 强化页面层级
2. 强化“进入栏目 / 翻开刊物”的感觉

## 10. Chat 的角色

chat 不再只是一个悬浮工具按钮，而是全局研究助理。

在新的信息架构里：

- 首页 chat 更偏“主专题解释”
- 日报页 chat 更偏“今天发生了什么”
- 专题页 chat 更偏“最近几天主题怎么演化”
- 档案页 chat 更偏“主题状态 / 公司位置 / 时间线解释”

本次不重做 chat agent 能力，只调整它在前端结构中的位置、感知与表现方式。

## 11. 技术实现边界

本次优先采用静态站点继续演进，不引入新的前端框架。

实现方式应优先：

- 复用现有模板渲染链
- 增加新的静态页面产物
- 保持 `report.json`、`theme_tracking_brief.json`、`theme_dossier.json` 等产物继续可复用

必要新增可以包括：

- 主刊首页专用模板
- 专题页模板
- 档案页模板
- 更清晰的导航数据装配层

不建议：

- 一次性重写全部 render 逻辑
- 引入复杂客户端路由
- 让页面结构依赖前端运行时状态才能成立

## 12. 降级策略

考虑到某些日期日报内容较少，页面结构应当稳定降级。

### 12.1 首页低产出日

当某日内容较少时：

- 首页仍由主专题驱动
- 最近几期入口补足内容感
- 不强依赖当天内容丰富度

### 12.2 专题页低内容日

如果当日无新主题推进：

- 专题页继续展示最近几天的专题脉络
- 保留“今天专题推进有限”的说明

### 12.3 档案页低内容日

如果 dossier 更新有限：

- 档案页维持主题定义、状态与时间线
- 不因当天无新条目而失去意义

## 13. 测试与验收

### 13.1 验证重点

实现后至少要验证：

- 首页、日报页、专题页、档案页都能稳定生成
- 导航路径正确
- 深度内容已从日报页中合理拆出
- 首页不会因低产出日报而显得空壳
- chat 入口在新结构下仍可用
- 移动端与桌面端都可读

### 13.2 视觉验收标准

达到以下标准才算成功：

1. 首页一眼看上去像“封面”，而不是“仪表盘”
2. 页面主次明显，不再所有区块都一样重
3. 日报页更轻、更快读
4. 专题和档案真正形成第二层和第三层阅读
5. 动效增强表现力，但不妨碍信息提取

## 14. 推荐实施顺序

建议按以下顺序实施：

1. 重构信息架构与新页面产物
2. 首页封面化
3. 深度内容从日报页拆出
4. 专题页与档案页落地
5. 统一导航
6. 加入轻量动效
7. 最后做前端 polish 与移动端修正

## 15. 结论

本次前端改造不是一次单纯的视觉美化，而是：

**从“单页情报面板”升级为“分层阅读的科技情报杂志”。**

明确的产品结构应为：

- 主刊首页：主专题封面
- 日报页：当期正文
- 专题页：深度栏目
- 档案页：主题沉淀
- Chat：全局研究助理

这套结构可以同时解决：

- 艺术感不足
- 表现力不足
- 单页过长
- 深度内容堆叠
- 首页缺少封面感

并为后续继续深化 `Research Assistant`、`Theme Dossier` 和更多专题能力提供更合适的前端承载方式。
