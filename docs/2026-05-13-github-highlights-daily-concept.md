# GitHub 今日 Highlight 日报构想

## 定位

这是科技巨头日报之外的一个 `兄弟日报`。

科技巨头日报关注：

- 官方信源
- 公司动作
- 产品、模型、战略与趋势判断

GitHub 今日 Highlight 日报关注：

- GitHub 当天或最近热度明显上升的项目
- 趋势仓库
- 高讨论度开源发布
- 值得快速关注的新工具、新框架、新 AI 项目

它不替代科技巨头日报，而是补充 “开发者侧、开源侧、项目侧” 的每日观察。

## 目标

产出一份面向个人快速浏览的日报，回答这些问题：

- 今天 GitHub 上最值得看的项目是什么
- 哪些项目正在快速升温
- 它们分别属于什么方向
- 哪些是短期热点，哪些可能值得持续追踪

## 可能的内容结构

### 1. 今日总览

- 日期
- 今日入选项目数
- 今日最热主题
- 一句话判断

### 2. 热门项目卡片

每个项目至少包含：

- 项目名
- 仓库链接
- 简要摘要
- 所属主题
- 热度信号
- 为什么值得看

### 3. 主题聚合

例如：

- AI agent
- coding assistant
- browser automation
- inference / serving
- data engineering
- frontend tooling

### 4. 值得持续跟踪

与“今天突然热”不同，这个区块更偏：

- 连续几天都在上升的项目
- 有真实落地潜力的仓库
- 可能进入后续主观察名单的项目

## 与当前项目的关系

建议把它设计成与科技巨头日报并列的 sibling pipeline，而不是直接硬塞进现有日报逻辑。

推荐原则：

- 保持独立配置
- 保持独立抓取源
- 保持独立分类与摘要策略
- 复用已有公共能力，例如：
  - LLM 表达层
  - 渲染层思路
  - 自动化、health-check、dry-run 能力

## 下一步建议

如果后续要正式开始做这条线，建议按下面顺序推进：

1. 明确 GitHub 热度信号来源
2. 定义项目入选规则
3. 定义页面结构
4. 设计独立数据模型
5. 再决定是否复用当前静态日报模板
